# Contributing

Thanks for your interest! Contributions are welcome.

## How to contribute

1. **Fork** this repository to your own GitHub account.
2. Create a branch for your change: `git checkout -b my-feature`.
3. Build and test on a device (Release config recommended — see the README).
4. Commit with a clear message and **open a Pull Request** against `main`.
5. A maintainer will review, discuss, and merge.

Please keep PRs focused — one feature or fix per PR is easiest to review.

## Ground rules

- Don't add Ableton (or any third party's) proprietary code, fonts, icons, or
  other assets. References to "Ableton Move" for hardware compatibility are fine;
  copied material is not. See the disclaimer in the README.
- Signing config is personal — don't hardcode your Apple Team ID into shared
  files; use your own local settings.

## Notes for maintainers

`main` is the public line. Work-in-progress and personal build branches live in a
separate private mirror and are not part of the public repo.
